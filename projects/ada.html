Overview

Ada is my personal AI assistant — designed to run locally, entirely offline. She listens, speaks, and automates using a hybrid of on-device inference and modular command pipelines. The goal was to create an AI that respects privacy, performs efficiently, and feels deeply integrated with the local system.

Core Architecture

  • Whisper.cpp — real-time speech-to-text on CPU.
  • Ollama — local LLM engine (Llama 3, Mistral, Gemma, etc.).
  • DuckDB — fast local vector store for semantic memory.
  • Python + Node.js bridge — to run CLI commands, automate tasks, and respond via TTS.

Example Workflow


User: "Ada, summarize my notes from today."
Ada → Transcribes audio → Generates embedding context via DuckDB → 
Calls Ollama locally → Responds with voice synthesis.

Features

  • 🎙️ Voice-based conversation using Whisper.cpp
  • 🧠 Local LLM reasoning with contextual memory
  • ⚙️ CLI automation for developer productivity
  • 🔒 Private by design — no external API calls
  • 🌐 Modular architecture for easy extensions (calendar, notes, search)

Technical Challenges

Running multiple models locally required careful optimization of memory and I/O. Ada uses lazy loading, streaming inference, and persistent DuckDB caching to maintain responsiveness on consumer hardware (Intel Arc GPU + M2 Mac).

Results

Ada can process natural voice commands in real time, execute local automation scripts, and provide conversational feedback without internet access. She’s a demonstration of local-first AI design — fast, private, and personal.

Stack & Tools

  • Python 3.11
  • FastAPI / Flask (API layer)
  • Node.js + Electron (UI prototype)
  • Ollama, Whisper.cpp, DuckDB
